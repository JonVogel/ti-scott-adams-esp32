TI BASIC Simulator — Test Programs
===================================
Copy-paste each section into the serial terminal and run with RUN.
Remember to type NEW before each new program.


=== 1. READ / DATA / RESTORE ===

NEW

10 FOR I=1 TO 5
20 READ A,B$

=== 28. RESEQUENCE / RES ===

Renumbers all program lines and updates GOTO/GOSUB/THEN/ELSE/RESTORE
targets so cross-references stay intact. Defaults: start=100, incr=10.

NEW
1 PRINT "LINE 1"
5 GOTO 37
15 PRINT "LINE 15 - SHOULD BE SKIPPED"
37 PRINT "JUMPED TO 37"
42 GOSUB 99
43 END
99 PRINT "SUB CALLED" :: RETURN
RES

After RES, LIST should show lines renumbered 100,110,... with GOTO
and GOSUB targets updated to match the new numbers:

100 PRINT "LINE 1"
110 GOTO 140
120 PRINT "LINE 15 - SHOULD BE SKIPPED"
130 END                 (wait — original 37 became 130? let's recount)

Actual renumber: 1→100, 5→110, 15→120, 37→130, 42→140, 43→150, 99→160
So LIST shows:
100 PRINT "LINE 1"
110 GOTO 130
120 PRINT "LINE 15 - SHOULD BE SKIPPED"
130 PRINT "JUMPED TO 37"
140 GOSUB 160
150 END
160 PRINT "SUB CALLED" :: RETURN

RUN output:
LINE 1
JUMPED TO 37
SUB CALLED

Optional: try `RES 10,10` to renumber starting at 10 with increment 10.

=== 30. SUB / SUBEND / SUBEXIT / CALL sub(args) ===

Subprogram support: numeric + string parameters, pass-by-value for
expressions and pass-by-value-result for bare variable arguments
(sub's final param value is copied back into the caller's variable
at SUBEND / SUBEXIT). No local scope yet — params leak into caller.
Nesting up to 8 deep. SUBs are typically placed at end of program;
main flow skips past them automatically.

NEW
5 CALL CLEAR
10 CALL GREET("ALICE")
20 CALL ADD(5,7,R)
30 PRINT "SUM=";R
35 X$="WORLD" :: CALL SHOUT(X$) :: PRINT X$
40 CALL MAYBE(0)
50 CALL MAYBE(1)
60 END
100 SUB GREET(NAME$)
110 PRINT "HELLO, ";NAME$
120 SUBEND
200 SUB ADD(A,B,C)
210 C=A+B
220 SUBEND
250 SUB SHOUT(S$)
260 S$=S$&"!"
270 SUBEND
300 SUB MAYBE(F)
310 IF F=0 THEN 330
320 PRINT "F IS TRUE"
330 IF F=0 THEN SUBEXIT
340 PRINT "AFTER CHECK"
350 SUBEND
SAVE Test30
RUN

Expected:
HELLO, ALICE
SUM= 12
WORLD!
F IS TRUE
AFTER CHECK

--- Part D: ON BREAK NEXT — ignore break ---

--- Part E: ON WARNING semantics ---
Divide-by-zero is a warning in TI XB: the evaluator substitutes ±MAX
(1.4E38) and signals the warning. ON WARNING controls what happens.

NEW
10 ON WARNING PRINT
20 A=1/0
30 PRINT "A=";A
RUN
(Expected:
  * WARNING: NUMBER TOO BIG IN 20
  A= 1.4E38 )

NEW
10 ON WARNING NEXT
20 A=1/0
30 PRINT "A=";A
RUN
(Expected: just `A= 1.4E38` — warning suppressed.)

NEW
10 ON WARNING STOP
20 A=1/0
30 PRINT "NOT REACHED"
RUN
(Expected: `* WARNING: NUMBER TOO BIG IN 20` then halt.)

--- Part F: ON WARNING STOP + ON ERROR traps upgraded warning ---
NEW
10 ON ERROR 100
20 ON WARNING STOP
30 A=1/0
40 PRINT "NOT REACHED"
100 PRINT "TRAPPED"
110 END
RUN
(Expected: `TRAPPED` — STOP promotes the warning to an error, and
ON ERROR jumps to the handler.)


=== 34. File I/O — OPEN / CLOSE / PRINT # / INPUT # / LINPUT # / EOF ===

  FLASH.NAME — internal LittleFS

INTERNAL, RELATIVE, FIXED are parsed and ignored.

--- Part A: write a file on FLASH ---

=== 44. Conway's Game of Life ===

Classic cellular automaton on a 24×32 toroidal grid (edges wrap).
Each generation applies the four rules:
  - alive cell with 2 or 3 live neighbors stays alive
  - alive cell otherwise dies (loneliness or overcrowding)
  - dead cell with exactly 3 live neighbors comes alive
  - everything else stays dead
The seed plants three classic patterns: a still-life "block" in the
upper-left, a period-2 "blinker" in the middle, and a glider that
walks down-and-right (and reappears at the top-left corner thanks
to the toroidal edges). Press any key to stop.

NEW
10 CALL CLEAR
15 CALL COLOR(2,16,1)
20 DIM A(24,32),B(24,32)
30 REM SEEDS: BLOCK, BLINKER, GLIDER
40 A(3,3)=1
41 A(3,4)=1
42 A(4,3)=1
43 A(4,4)=1
50 A(10,10)=1
51 A(10,11)=1
52 A(10,12)=1
60 A(15,6)=1
61 A(16,7)=1
62 A(17,5)=1
63 A(17,6)=1
64 A(17,7)=1
70 REM INITIAL PAINT
80 FOR R=1 TO 24
81 FOR C=1 TO 32
82 IF A(R,C)=0 THEN 90
85 CALL HCHAR(R,C,42)
90 NEXT C
91 NEXT R
100 REM COMPUTE NEXT GENERATION
110 FOR R=1 TO 24
111 FOR C=1 TO 32
120 N=0
130 FOR DR=-1 TO 1
131 FOR DC=-1 TO 1
140 IF DR<>0 THEN 150
141 IF DC=0 THEN 180
150 RR=R+DR
151 IF RR<1 THEN RR=RR+24
152 IF RR>24 THEN RR=RR-24
160 CC=C+DC
161 IF CC<1 THEN CC=CC+32
162 IF CC>32 THEN CC=CC-32
170 N=N+A(RR,CC)
180 NEXT DC
181 NEXT DR
190 B(R,C)=0
191 IF A(R,C)<>1 THEN 195
192 IF N=2 THEN B(R,C)=1
193 IF N=3 THEN B(R,C)=1
194 GOTO 200
195 IF N=3 THEN B(R,C)=1
200 NEXT C
201 NEXT R
210 REM APPLY DELTAS
220 FOR R=1 TO 24
221 FOR C=1 TO 32
230 IF B(R,C)=A(R,C) THEN 250
240 IF B(R,C)=1 THEN CALL HCHAR(R,C,42) ELSE CALL HCHAR(R,C,32)
241 A(R,C)=B(R,C)
250 NEXT C
251 NEXT R
260 CALL KEY(0,K,S)
261 IF S<1 THEN 110
270 END
RUN

Expected:
- Block (top-left) sits unchanged forever.
- Blinker (middle) flips between horizontal and vertical every gen.
- Glider walks diagonally down-right, wraps at the edges.

Generation rate is BASIC-bound (~6000 array reads + 768 cell
checks per gen). On the ESP32 fast-path that's a few generations
per second — fast enough to watch the glider crawl.
